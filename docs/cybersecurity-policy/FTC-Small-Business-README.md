# FTC: Cybersecurity for Small Business - Basics

**Source:** U.S. Federal Trade Commission (FTC)  
**Date Added:** April 10, 2026  
**Location:** `docs/cybersecurity-policy/FTC-Cybersecurity-Small-Business-Basics.pdf`  
**Reference:** [FTC.gov/SmallBusiness](https://ftc.gov/SmallBusiness)

## Overview

U.S. government guidance on cybersecurity fundamentals for small businesses. This resource provides practical, actionable recommendations aligned with the NIST Cybersecurity Framework.

## Core Recommendations

### 🔐 Protect Your Files & Devices

| Action | Details |
|--------|---------|
| **Update software** | Apps, browsers, OS - set to auto-update |
| **Back up files** | Offline (external drive), cloud, or both |
| **Secure paper files** | Physical document security matters too |
| **Require passwords** | All laptops, tablets, smartphones |
| **Encrypt devices** | Laptops, tablets, phones, removable drives, backup tapes, cloud storage |
| **Multi-factor authentication** | Required for sensitive areas of network |

### 📶 Protect Your Wireless Network

- **Change default router name and password**
- **Turn off remote management**
- **Log out as administrator** after setup
- **Use WPA2 or WPA3 encryption** - protects data in transit from outsiders

### 👥 Smart Security Practices

| Practice | Implementation |
|----------|----------------|
| **Strong passwords** | 12+ characters, mix of numbers/symbols/case |
| **No password reuse** | Never reuse or share via phone/text/email |
| **Limit login attempts** | Prevents password-guessing attacks |
| **Staff training** | Regular schedule, block access if not completed |
| **Breach response plan** | Save data, run business, notify customers |

## NIST Cybersecurity Framework (5 Functions)

The FTC guide aligns with the **NIST Cybersecurity Framework** - voluntary best practices for businesses of all sizes.

### 1. IDENTIFY 📋
- Inventory all equipment, software, and data (laptops, phones, tablets, POS devices)
- Create and share company cybersecurity policy covering:
  - Roles and responsibilities for employees, vendors, anyone with sensitive data access
  - Steps to protect against attacks and limit damage

### 2. PROTECT 🛡️
- Control network logins and device access
- Use security software to protect data
- Encrypt sensitive data (at rest and in transit)
- Conduct regular backups
- Update security software (automate if possible)
- Formal policies for disposing of electronic files and old devices
- **Train everyone** on cybersecurity (personal risk + workplace role)

### 3. DETECT 🔍
- Monitor computers for unauthorized access, devices (USB), and software
- Check network for unauthorized users/connections
- Investigate unusual activities

### 4. RESPOND 🚨
Have a plan for:
- Notifying customers, employees, and affected parties
- Keeping operations running
- Reporting to law enforcement/authorities
- Investigating and containing the attack
- Updating policies with lessons learned
- Preparing for emergencies (weather, etc.)
- **Test your plan regularly**

### 5. RECOVER ♻️
After an attack:
- Repair and restore affected equipment/network
- Keep employees and customers informed of response/recovery activities
